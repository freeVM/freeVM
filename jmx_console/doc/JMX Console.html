About this document

Purpose

This document introduces the DRL implementation of the Java* Management Extensions (JMX*) Console. The document gives details on the JMX* Console design, includes an overall description of the package and points out basic user's actions while dealing with the two versions of the implementation.

Intended Audience

The target audience for the documentation includes a wide community of engineers interested in using the JMX* Console in their applications and in future work with the product to contribute to its development. The document assumes that the readers are familiar with the JMX* technology and the Java* programming language.

Documentation Conventions

This document uses the unified conventions for the DRL documentation kit.

Introduction to the JMX* Console

JMX* introduces a new sort of applications, called manageable. This invention effects many aspects of software design and development. The JMX* application is easy-manageable due to the control panel, a so called console, showing the status of the application and distributing commands. The JMX* console enables management of all JMX* compliant systems and provides flexible and intuitive interface for users.

JMX* Console in DRL

The DRL JMX* console is the application intended for managing of various JMX* compliant systems.

About

The JMX* console is an open-source clean-room tool targeted at managing Java* objects, so called MBeans, registered in the MBean server. For more details on JMX* refer to [1] and [2].

The JMX* console exists in two versions provided in the current implementation:

JMX* Console Version Compatibility
Standalone Standard Widget Toolkit (SWT) SWT 3.1.1
Eclipse* plug-in Eclipse* SDK, versions 3.1.0 and 3.1.1

You may use the console with other versions of SWT and Eclipse* SDK, but success is not guaranteed.

Inside the JMX* Console

The JMX* console is developed according to the Model View Controller (MVC) architectural pattern and consists of the following components:

  • Model represented by the MBean server
  • View represented by the org.apache.harmony.x.management.console.standalone package when used as the standalone console and by the org.apache.harmony.x.management.console.plugin package when used as the plug-in
  • Controller represented by the org.apache.harmony.x.management.controller package

The MBean server is not described in the document. For information on it see [4].

Controller

The controller interface serves as an abstraction from the JMX* semantics. This component acts just like the MBean Server, but makes its activities fully transparent and, therefore, requires no knowledge of JMX* specifics. This can save some of your time.

The Controller can connect to any JMX* compliant server and expose its content. The interface and its supporting classes are designed in a handy way for data presentation in the Graphical User Interface (GUI).

The Controller provides a generalized view of the JMX* server. This interface presents the registered MBeans as objects with attributes, operations and notifications without distinguishing them by their special meanings, if there are any, to the MBean Server.

Another view is a specific MBean server "vision" of certain MBeans, such as the MLet Service or the relation service. Note that this feature is only declared in the interface and has no specific implementation in the current version of the JMX* console.

For details on the Controller, see the API documentation of the package org.apache.harmony.x.management.controller.

User Interface Extension

The user interface is an Eclipse* plug-in using the Controller to obtain, send and receive data from the MBean Server, to execute operations and to obtain notifications. Currently, the user interface has two aspects for the further development and extension: to add new Eclipse* views depending on the MBeanTreeView class and to add attribute editors to show or edit the attribute of a new type.

New Eclipse* View Implementation

The standard procedure of extending the Eclipse* plug-in is to create a new extension at a certain extension point. To extend the JMX* console, use the org.eclipse.ui.views point [5].

Note

Further in the document, this extension and the class corresponding to it are referred to as view.

The view class extends the org.eclipse.ui.part.ViewPart class and implements the org.apache.harmony.x.management.ui.IMBeanTreeDependant interface. This enables the view class to receive callbacks from the MBean tree while browsing it. For more information, see the API documentation of the org.apache.harmony.x.management.ui package and the implementation patterns used in the org.apache.harmony.x.management.ui.AttributeView class.

New Attribute Editor Implementation

The attribute editors are located in the package org.apache.harmony.x.management.ui.editor. Currently, the implementation has certain editors for individual attribute types:

  • BooleanEditor for boolean attributes
  • NumericEditor for numbers
  • ArrayEditor for arrays
  • CommonEditor for all other types

To create a new attribute editor for the type not covered by the aforementioned editors, extend AbstractEditor. For details, refer to the API documentation of the org.apache.harmony.x.management.ui.editor package and follow the implementation pattern from the org.apache.harmony.x.management.ui.editor.NumericEditor class.

How to Use the JMX* Console

This section describes two basic scenarios of using the two versions of the JMX* console implementation: how to start, connect and use the JMX* console. From this part of the document, you will get certain information about usage of the console. After performing all the operations described below, you will be able to experience all the advantages of the JMX* console.

Building and Installing the JMX* Console

To start working with the JMX* console, follow the instructions below according to the implementation version in use:

JMX* Console Plug-in Standalone JMX* Console
1. Install Eclipse* SDK 3.1.1 into a folder <ECLIPSE_HOME>. 1. Install SWT 3.1.1 into a folder <SWT_HOME>.
2. Unzip the JMX* console bundle into a folder <CONSOLE_HOME>. 2. Unzip the JMX* console bundle into a folder <CONSOLE_HOME>.
3. Change the current directory to <CONSOLE_HOME>/Harmony/make. 3. Change the current directory to <CONSOLE_HOME>/Harmony/make.
4. Build the plug-in console .jar file running the following command: 
ant jar.plugin
4. Build the standalone console .jar file running the following command: 
ant jar.pluginant  jar.standalone
5. To install the implementation, copy org.apache.harmony.x.management.console_3.1.1.jar from <CONSOLE_HOME>/Harmony/build/jar/ into the plugins/ folder and restart Eclipse*.

As the result, the new JMX* perspective will become available.

    For the plug-in mode, using the JMX* Connection Wizard select:
    File > New > Other... > JMX Console > New Connection Wizard

    For the standalone mode, select:
    Connection > New Connection

    Figure 2: Starting the JMX* Connection Wizard

  2. Select one of three modes appropriate for your purpose.

Managing MBeans

After the console is connected to the MBean server, the view containing the tree of all MBeans registered in this MBean server appears on the screen. The domains of MBeans are the branches of the first layer and MBeans themselves are branches of the second layer. If you select any MBean, the information about it is shown in the Attribute and Operation views.

JVM* is the classical manageable application, which starts the internal MBean server and registers a number of MBeans in it [3] .

Managing Attributes

Three types of the MBean attributes exist: readable, writable or both. The read-only attributes are shown in the Attributes view as a grayed text box and can be copied and pasted into other applications, but cannot be edited. Readable and writable attributes can undergo all kinds of editing. If the value of the attribute is large, the Collapse/Expand button appears after the name of the attribute.

Managing Notifications

The interface of the MBean Notification Table consists of the following elements:

  • The checkbox indicating whether you are subscribed to notifications emitted by the selected MBean or not
  • The Clear List button that can remove all received notifications
  • The Update button that reads all received notifications and shows them in the table
