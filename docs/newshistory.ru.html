5.0 - M2 Apache Harmony Доступна новая стабильная сборка
Мы рады сообщить, что проект Apache Harmony выпустил новую стабильную версию Java-платформы 1.5.0, версия М2. В этой сборке улучшена общая стабильность кода, в качестве коллектора по умолчанию используется GCv5, добавлено множество оптимизаций для JIT-компилятора, заметно улучшено использование памяти, исправлено большое количество ошибок, а покрытие кода достигло 99.06%.
Новую сборку можно скачать здесь Отзывы вы можете присылать на адрес почтовой рассылки проекта. (2007-06-30)
Доступны стабильные сборки кода на платформе Windows x86-64
Теперь вы можете воспользоваться готовой сборкой кода Harmony для платформы Windows на архитектуре x86-64, см. подробную информацию о предлагаемых сборках на странице для скачивания. (2007-05-14)
Доступна первая стабильная сборка Apache Harmony
Мы рады предложить вам первую стабильную сборку версии 5.0, ранее доступную под ревизией кода r535500. Эта версия прошла дополнительное тестирование и наиболее стабильна. Вы можете скачать и запустить и среду исполнения JRE, и комплект разработчика HDK. Об ошибках и пожеланиях просим сообщать через почтовую рассылку проекта. (2007-04-30)
Выпущена виртуальная машина JRockit с использованием библиотек классов Harmony
Компания BEA объявила о выпуске пробной версии виртуальной машины JRockit, третьей по популярности Java виртульной машины в мире, которая может использовать библиотеки классов Apache Harmony. Новая версия машины доступна по пробной лицензии в бинарном виде. Виртуальная машина JRockit не только продемонстрировала возможности портирования библиотек классов Harmony, но и пополнила список приложений, способствующих созданию независимой реализации Java-платформы. (2007-01-30)
Доступна сборка комплекта разработчика JDK
Проект Apache Harmony теперь предлагает комплект разработчика (Java development kit), включающий такие инструменты javac, keytool и другие. Работа в этом направлении еще не завершена, но выпуск этой сборки - первый шаг на пути формирования стандартного традиционного поставляемого комплекта инструментов разработчика. Подробности о предлагаемых сборках кода см. на странице для скачивания. (2006-12-27)
Доступны стабильные сборки кода на 64-битной платформе Linux x86-64

Apache Harmony стал проектом верхнего уровня сообщества Apache Software Foundation
Проект Apache Harmony теперь входит в число полномасштабных проектов сообщества Apache Software Foundation. 25 октября Совет Директоров принял решение включить проект Harmony в число верхнеуровневых проектов, а 29 октября комитет управления проектом Apache Incubator проголосовал за перевод проекта Harmony на новый уровень по результатам выполенения требований Сообщества. В настоящее время мы находимся в процессе перевода ресурсов и инфраструктуры на новые мощности, приносим извинения за причиненные неудобства. (2006-10-29)
Доступны новые сборки кода
В новые сборки кода вошел целый ряд исправлений и добавлений, включая новый коллектор и практически полную реализацию интерфейса JVMTI. Кроме того, виртуальная машина IBM VME теперь может функционировать в структуре среды исполнения JRE. Набор библиотек продолжает пополняться новыми классами, в том числе CORBA, за что мы благодарим проект Apache Yoko. В этой версии сборки выполнено множество исправлений и поправок библиотеках классов.
Воспользуйтесь сборкой среды исполенния для тестирования работы Вашего приложения или сборкой комплекта инструментов разработчика для того, чтобы способствовать развитию библиотек классов Harmony. Свои отзывы о работе сборки Вы можете публиковать через рассылку. (2006-09-28)

Доступны новые сборки кода
Проект Harmony представляет готовые сборки среды исполнения и комплекта инструментов разработчика. Отметим, что эти сборки не являются стабильным, полным или совместимым программным продуктом. Однако, вы можете использовать этот код для исполнения Ваших Java приложений и тем самым способстовать поиску и исправлению ошибок, направленному на создание полной реализации основных пакетов библиотек классов. Сборка кода комплекта инструментов создана для разработчиков и включает архивные jar файлы, внутренние библиотеки и скрипты для сборки.(2006-07-22)

Реализация кода виртуальной машины DRL Virtual Machine

Реализация кода Java Security
30 декабря 2005 года проект Apache Harmony получил существенный вклад в виде реализации кода еще одного пакета библиотек. Вклад был сделан Михаилом Лоенко от корпорации Интел. Новый код включает реализацию пакетов защиты, криптографии, пакета javax.net и набор тестов. Выражаем благодарность Михаилу и его коллегам в подразделении серверного ПО и решений корпорации Интел. (2006-01-16)
Реализация интерфейса виртуальной машины и основных классов
Компания IBM предложила в пользование проекту Apache Harmony набор основных Java классов и реализацию интерфейса между библиотеками классов и виртуальной машиной. Тим Эллисон прокомментировал:
"Я польщен возможностью сделать вклад в развитие проекта Harmony от имени IBM. Новый код включает конкретную реализацию интерфейса между виртуальной машиной и библиотеками классов, которую мы недавно обсуждали, вместе с набором основных классов".
Нового кода достаточно, чтобы вместе с приложением Apache Ant и Java компилятором от Eclipse обеспечить функционирование базовой независимой среды исполнения. Компания IBM также открыла доступ к своей реализации виртуальной машины J9 VM для того, чтобы проект смог опробовать новый код. Слова благодарности Тиму и компании IBM :) (2005-11-08)
Реализация "загрузочной виртуальной машины"
Проект Apache Harmony получил вклад от Дэниела Лидикто в виде реализации базовой Java виртуальной машины, которая называется "Загрузочная Виртуальная Машина Apache Harmony". Дэн предоставил этот код чтобы:
  • Сформировать основу для построения виртуальных машин
  • Создать структуру загрузки и исполнения для будущей "продуктовой" версии виртуальной машины
  • Привлечь интерес к обсуждению архитектуры основных компонентов среды
Слова благодарности Дэну! (2005-09-30)
Исходные код виртуальной машины JCVM
