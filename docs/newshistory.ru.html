Apache Harmony 5.0 M9 Доступна новая стабильная сборка
Мы рады сообщить, что проект Apache Harmony выпустил новую стабильную версию Java-платформы 1.5.0, версии М9.

В эту сборку вошли многочисленные исправления ошибок, а также следующие улучшения:

  • производительность: разнообразные оптимизации в библиотеках классов и JIT компиляторе;
  • pack200: новая реализация технологии JSR-200 упаковки байткодов Ява;
  • началось портирование библиотек классов на системы AIX и zOS;
  • исправление ошибок: фокус на успешное прохождение Eclipse Test Suite и улучшение нашей системы тестов.

Apache Harmony 5.0 M7 Доступна новая стабильная сборка
Проект Apache Harmony выпустил новую стабильную сборку Java-платформы 1.5.0, версии M7. В эту версию включены многочисленные улучшения:
  • новые оптимизации производительности, особенно в области сериализации объектов и распаковщика unpack200
  • новый дизассемблер класс-файлов 'javap', альфа-версия настройщика безопасности 'policytool'
  • исправления ошибок и функциональные улучшения по всей библиотеке классов, особенно в Swing widget toolkit
  • ...и многое другое!

Мы бы хотели отдельно поблагодарить наших участников Google Summer of Code: András Belicza (policytool) и Tharindu Mathew (Swing enhancements) - и ещё раз приветствовать их в проекте!

Эта сборка распространена на сайтам загрузки, и мы настоятельно рекомендуем всем пользователям обновиться до версии Apache Harmony 5.0M7.

Исходный код и сборки проекта можно скачать здесь. Отзывы вы можете присылать на адрес почтовой рассылки проекта. (2008-18-28)

5.0 - М3 Apache Harmony Доступна новая стабильная сборка
Эта версия направлена на повышение общей стабильности и быстродействия среды. В новую сборку вошли следующие улучшения: инновационная методика быстрой проверки байтового кода, отложенное разрешение символьных ссылок, и поддержка Java-классов версии 6.0. Кроме того, были усовершенствованы следующие функциональные области:
  • Отладочная информация на уровне байткода в JIT-компиляторе - более информативные сообщения о стеке при возникновении исключений
  • Параллельность - частично изменена поддержка поточности для улушчения надежности и удобства поддержки кода
  • Отладка в режиме JVMTI с помощью JIT-компилятора - увеличение скорости в 3 раза
  • Стабильность кода модулей Swing и AWT на ОС Windows
Новую сборку можно скачать здесь. Отзывы вы можете присылать на адрес почтовой рассылки проекта. (2007-10-09)

Apache Harmony стал проектом верхнего уровня сообщества Apache Software Foundation
Проект Apache Harmony теперь входит в число полномасштабных проектов сообщества Apache Software Foundation. 25 октября Совет Директоров принял решение включить проект Harmony в число верхнеуровневых проектов, а 29 октября комитет управления проектом Apache Incubator проголосовал за перевод проекта Harmony на новый уровень по результатам выполенения требований Сообщества. В настоящее время мы находимся в процессе перевода ресурсов и инфраструктуры на новые мощности, приносим извинения за причиненные неудобства. (2006-10-29)

Доступны новые сборки кода
Проект Harmony представляет готовые сборки среды исполнения и комплекта инструментов разработчика. Отметим, что эти сборки не являются стабильным, полным или совместимым программным продуктом. Однако, вы можете использовать этот код для исполнения Ваших Java приложений и тем самым способстовать поиску и исправлению ошибок, направленному на создание полной реализации основных пакетов библиотек классов. Сборка кода комплекта инструментов создана для разработчиков и включает архивные jar файлы, внутренние библиотеки и скрипты для сборки.(2006-07-22)

Реализация интерфейса виртуальной машины и основных классов
Компания IBM предложила в пользование проекту Apache Harmony набор основных Java классов и реализацию интерфейса между библиотеками классов и виртуальной машиной. Тим Эллисон прокомментировал:
"Я польщен возможностью сделать вклад в развитие проекта Harmony от имени IBM. Новый код включает конкретную реализацию интерфейса между виртуальной машиной и библиотеками классов, которую мы недавно обсуждали, вместе с набором основных классов".
Нового кода достаточно, чтобы вместе с приложением Apache Ant и Java компилятором от Eclipse обеспечить функционирование базовой независимой среды исполнения. Компания IBM также открыла доступ к своей реализации виртуальной машины J9 VM для того, чтобы проект смог опробовать новый код. Слова благодарности Тиму и компании IBM :) (2005-11-08)
Реализация "загрузочной виртуальной машины"
Проект Apache Harmony получил вклад от Дэниела Лидикто в виде реализации базовой Java виртуальной машины, которая называется "Загрузочная Виртуальная Машина Apache Harmony". Дэн предоставил этот код чтобы:
  • Сформировать основу для построения виртуальных машин
  • Создать структуру загрузки и исполнения для будущей "продуктовой" версии виртуальной машины
  • Привлечь интерес к обсуждению архитектуры основных компонентов среды
Слова благодарности Дэну! (2005-09-30)
