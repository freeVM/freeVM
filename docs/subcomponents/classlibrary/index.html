Apache Harmony Class Library

The following documentation links are specific to the class library effort underway at Apache Harmony.

Developing and Building the code
  • Building the Classlibrary Code

    Describes how to download and build the class library code from source.

  • Development with the Eclipse IDE

    Describes how to set up Eclipse to develop the class library code.

  • Harmony Development Kit

    Proposed description of a Harmony Development Kit, which enables a developer to rebuild a standalone classlib module.


Project Conventions
  • Java package naming guidelines

    Guidelines for Harmony package naming convention.

  • Testing conventions in the Apache Harmony Classlib

    Proposed placement and package naming conventions for different types of Harmony class library tests.

  • Compatibility guidelines

    Proposed guidelines for ensuring that the Harmony class library is compatible with other Java implementations.

  • Framework for testing serialization

    Proposed approach for testing serialization compatibility with other certified implementations, Harmony backward compatibility, placement and naming conventions for resource files.

  • Harmony-dev agreements

    Summary of agreements and recommendations worked out on harmony-dev mailing list.


Architecture and guides
  • Class library porting documentation

    A porting guide aimed at developers interested in porting the classlib code to a new VM.

  • The Harmony class library port layer

    A description of the functions in the class library portlayer aimed at developers who are writing JNI code for use by the Harmony class libraries and developers who are interested in porting the classlib code to a new operating system.
