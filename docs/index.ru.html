Добро пожаловать в проект Apache Harmony!

RU | EN

Внимание: Данная страница является переводом английской версии на сайте и находится в стадии разработки.

Мы приглашаем всех желающих принять участие в разработке высококачественной сертифицированной реализации стандартной версии Java-платформы (Java 2 Standard Edition, J2SE). Целью проекта является развитие обширного активного сообщества разработчиков, заинтересованных в развитии сред исполнения. В задачи проекта входит создание:

  • совместимой независимой реализации платформы и инструментов Java-разработчика (J2SE JDK) под лицензией Apache, версия 2
  • модульной архитектуры платформы (виртуальной машины и библиотек классов), разработанной в рамках сообщества

Мы планируем портировать Apache Harmony на различные платформы. Поддержка определенной платформы зависит от готовности участников сообщества регулярно тестировать работу кода, публиковать результаты тестирования, искать и исправлять ошибки и т.д. Таблица на странице целей проекта содержит список поддерживаемых аппаратных платформ и операционных систем.

Для того, чтобы начать использовать Apache Harmony или принять участие в разработке этого проекта, вы можете воспользоваться следующими ресурсами:

  • Инструкция по использованию Apache Harmony
  • Инструкция по сборке и использованию исходного кода Apache Harmony

Новости проекта

Apache Harmony 5.0 M8 Доступна новая стабильная сборка
Мы рады сообщить, что проект Apache Harmony выпустил новую стабильную версию Java-платформы 1.5.0, версии М8.

В эту сборку вошли многочисленные исправления ошибок, а также улучшения в policytool и поддержке pack200.

Исходный код и сборки проекта можно скачать здесь. Отзывы вы можете присылать на адрес почтовой рассылки проекта. (2008-11-20)

Apache Harmony 5.0 M7 Доступна новая стабильная сборка
Проект Apache Harmony выпустил новую стабильную сборку Java-платформы 1.5.0, версии M7. В эту версию включены многочисленные улучшения:
  • новые оптимизации производительности, особенно в области сериализации объектов и распаковщика unpack200
  • новый дизассемблер класс-файлов 'javap', альфа-версия настройщика безопасности 'policytool'
  • исправления ошибок и функциональные улучшения по всей библиотеке классов, особенно в Swing widget toolkit
  • ...и многое другое!

Мы бы хотели отдельно поблагодарить наших участников Google Summer of Code: András Belicza (policytool) и Tharindu Mathew (Swing enhancements) - и ещё раз приветствовать их в проекте!

Эта сборка распространена на сайтам загрузки, и мы настоятельно рекомендуем всем пользователям обновиться до версии Apache Harmony 5.0M7.

Исходный код и сборки проекта можно скачать здесь. Отзывы вы можете присылать на адрес почтовой рассылки проекта. (2008-18-28)

Jikes RVM 3.0 + Apache Harmony!
Ian Rogers недавно анонсировал отличные новости - JikesRVM теперь поддерживает библиотеку классов Harmony:

"We are pleased to announce the release of Jikes RVM 3.0, an open-source meta-circular JVM written in Java. A new feature of the release is initial support for Apache Harmony's class library."

Для просмотра полного текста анонса и инструкций, как объединить Harmony и JikesRVM, обращайтесь в пост Ian'а. (2008-08-27)

Пожертвование кода Microemulator
Apache Harmony расширила покрытие спецификации на Java API! Мы приняли большое пожертвование исходного кода от авторов Microemulator.

Microemulator - Java-реализация Java ME API на Java SE. При помощи Microemulator становится возможным запуск мидлетов (MIDP/CLDC) как самостоятельных Java-приложений или апплетов.

Спасибо Бартеку (Bartek), Владу (Vlad), и Михаэлю (Michael) за это пожертвование! (2008-06-09)

Статус проекта

Проект Apache Harmony планомерно прогрессирует:

  1. Доступны последние сборки среды исполнения (JRE), платформы (JDK) и комплекта разработчика (HDK)
  2. Автоматически обновляется статус стабильности кода на всех поддерживаемых платформах
  3. В нашей реализации среды исполнения JRE успешно работают такие популярные приложения как Apache Tomcat, Eclipse, Maven, Derby, и Ant
  4. Реализовано более 99% интерфейса Java 5 (завершено но не сертифицировано на совместимость)
  5. Общий объем кода превысил 1.25 миллиона строк
