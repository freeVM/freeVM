Stable builds

We have just started to create stable builds: these are builds of the most recent tested code provided for your convenience. They have been declared best-so-far but they are not official releases of the Apache Harmony project or the Apache Software Foundation. Here you can find schedule and requirements for M2 milestone.

Back to top

M2 Schedule

We are planning to deliver stable builds each two months. Below is the schedule for the M2 stable build

  • 30-Jun-2007: M2 release
  • 23-Jun-2007: M2 code freeze. Only critical bug fixes go in.
  • 13-Jun-2007: M2 feature freeze. No major redesigns since that date.

M2 Requirements - Draft. Don't use the names please until the requirements are final

Each stable build is intended to have best achived stability, perfomance, compatibility, and completeness. The requirements below set specific focus for M2. Pass rates are measured for knowingly valid tests.

  • M2_TEST1: 98% pass rate for class libraries tests; no regression since M1
  • M2_TEST2: 98% pass rate for drl vm tests; no regression since M1
  • M2_TEST3: 98% pass rate for jdk_tools tests; no regression since M1
  • M2_TEST4: 98% pass rate of VM validation test suite - HARMONY-3206; no regression since M1
  • M2_TEST5: 98% pass rate of Functional test suite - HARMONY-3528; no regression since M1
  • M2_TEST6: 98% pass rate of Stress tests - HARMONY-3536; no regression since M1
  • M2_TEST7: 98% pass rate of Reliability test - HARMONY-2918; no regression since M1
  • M2_TEST8: 98% pass rate of Eclipse 3.2 unit tests; no regression since M1
  • M2_TEST9: 98% pass rate of Geronimo unit tests; no regression since M1
  • M2_TEST10: 98% pass rate of Eclipse TPTP profiler tests; no regression since M1
  • M2_APP1: Harmony works smoothly with Eclipse 3.2; no known bugs; no regression since M1
  • M2_APP2: Run Eclipse 3.2 Hello World Application scenario (link)
  • M2_APP3: Run HDK self-hosting (link)
  • M2_APP4: Run Simple Tomcat scenario (link)
  • M2_APP5: Run Apache Axis scenario (link)
  • M2_APP6: Run Apache Struts scenario (link)
  • M2_APP7: Run Jetty scenario (link)
  • M2_LONG1: 48 hour Eclipse scenario (link)
  • M2_PERF1: All Dacapo benches correctly with all workloads, new features or patches that will be commit prior M2 do not introduce degradations
  • M2_PERF2: Scimark benchmark works correctly, new features or patches that will be commit prior M2 do not introduce degradations
