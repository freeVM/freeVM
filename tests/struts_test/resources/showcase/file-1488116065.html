6:     <h1>Token Examples</h1>
7: 
8:     These examples illustrate Struts build in support of using tokens to prevent double post.
9:     <p/>
10:     You have a web page where user can input data and press a button to submit.
11:     There could be a problem that the user submit the data many times, by either clicking the
12:     button many times, or use the browser back button and then submit the form again.
13:     <br/> A good solution is to use a hidden token in the form. The token is autogenerated and unique
14:     from time to time. This token is then validated with the HttpSession that it is the first time
15:     it is submitted, if not we have a double post and therefore can prevent the second submit action.
16:     <p/>
17:     For more information check out javadoc for org.apache.struts2.interceptor.TokenInterceptor
18:     and org.apache.struts2.interceptor.TokenSessionStoreInterceptor.
